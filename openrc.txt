Последовательность загрузки
После инициализации ядра поднимается init процесс который:
	-проверяет монтирование FS;
	-выполняет сценарии из /etc/init.d/ которые запускают службы;
	-запускает сеть, Х-сервер и т.д.
	-поключает терминалы со службой agetty.


Запуск осуществляется из каталога /etc/init.d но не все что там находится не обязательно будет запушенно:
	- запускаются то из /etc/init.d, что имеет символьные ссылки из /etc/rulevels/boot
	- запуск процессов имеющих символьные ссылки из /etc/runlevels/default/

/etc/inittab - конфигурационный файл init службы.
	si::sysinit:/sbin/rc sysinit - инициализирует скрипт /sbin/rc который поднимает систему
	
	rc::bootwait:/sbin/rc boot - обращается к символьным ссылкам из /etc/runlevels/boot/

	id::3:initdefault - устанавливает уровень запуска по умолчанию. (3) 

	.
	c1:12345:respawn: /sbin/agetty 23400 tty3 linux - включение виртул консолей с заданными параметрами.
	.


Уровень запуска это состояние по умолчанию в котором запускается система. В gentoo имеется 7 уровней запуска: 3 служебные
	sysinit, shutdown, reboot
 и 4 пользовательские:
	boot, default,nonetwork,single - им соответствуют определенные каталоги из /etc/runlevels

Cценарии запускает rc процесс. Каждый сценарий имеет аргументы:
	start,stop, restart - запуск, остановка, перезапуск службы;
	[/etc/init.d/postfix start]

	status - показывает текущее состояние службы;
	zap - позволяет сбросить аргумент состояния службы. К примеру, служба запущенна - а в системе остановленна;
	ineed - показывает службы которые необходимы для использования данной службы;
	iuse - показывает возможные службы которые нужны нашей службе;
	needsme, usesme - показывает службы которые нуждаются в этой службе или возможно нуждаются
	broken - список служб необходимой для данной, но отсутствующий в системе.



rc-update del(..add,show..) postfix default - удаляет() сценарий из определенного уровня запуска

rc-update -v show - показывает список всех доступных сценариев. Без ключа показывает запущенные скрипты.


Для каждого сценария есть конфиг файл в /etc/conf.d с параметрами которые необходимы для запуска.
Если установить приложение не через portage, то скорее всего необходимо будет настраивать init вручную. Сценарии gentoo не
совместими со сценариями других дистрибутивах. Сценарии могут писаться в ручную. 

Можно посмотреть содержимое скрипта и увидеть необходимое для его запуска nano /etc/init.d/dhcpcd

Возможно также создать дупликат уровня default и выбирать один из них при работе с системой.
Включить логи rc нужно в файле /etc/rc.log rc_logger="YES"  затем журналы будут хранится в /var/log/rc.log

требуется заменить rc на openrc в inittab
а затем в скриптах все пильнуть runscript на openrc-run
 find /etc/init.d -type f -exec sed -i 's/runscript/openrc-run/' {} +
