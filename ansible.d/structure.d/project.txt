=== Структура проекта:

- ansible.cfg 		- параметры запуска ansible
- hosts/inventory.yml 	- машины (host, group_vars, host_vars для стейджа)
- group_vars		- переменные для группы (all.yml здесь)
- host_vars		- переменные для машин
- playbook.yml 		- основной файл для запуска
- playbook.d 		- папка содержит импорты из основного файла
- roles			- подключение ролей (путь может быть переоределен)


== Организация проекта
Деление проекта на 3-4 слоя:
1. Имя, дата, сеть, пользователи, права, фаервол
2. Установка приложений и их настройка (user_add возможно с шифрованием)
...

== Порядок обработки переменных
- Переменные накладываются (неожиданный результат) 
- Переменные перезаписывают друг друга, конечная точка содержит в себе все
  (дублирование)
